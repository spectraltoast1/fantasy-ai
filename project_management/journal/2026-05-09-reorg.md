## Decisions made

- **V1 = interview-derived strategy doc + advisor (start/sit at minimum) + dashboard with shared data layer.** → captured in PROJECT_OVERVIEW.md V1 success criteria
- **Transcript synthesis frozen for v1, preserved at `_deferred/synthesis_pipeline/`.** Includes the four pass prompts (FIRST through THIRD + MERGE), the token vocabulary template, the raw transcripts, the Pass 1 caches, the v1 truncated strategy MDs, and the pipeline scripts. → captured in PROJECT_OVERVIEW.md non-goals + that folder's README
- **The dashboard is a peer product to the AI, not a viewer.** At least one analytical panel must work without invoking the advisor. → captured in PROJECT_OVERVIEW.md project values
- **Time-series snapshots and the advisor output log are Phase 2 work.** Without them, the dashboard's trend views and history views can't exist. The data layer is shared between AI and dashboard. → captured in TECHNICAL_ARCHITECTURE.md
- **Pre-filtering context (instead of dumping full JSON) is Phase 3 work.** Current `advisor.py` violates the "pre-filter deterministically" design principle — Phase 3 fixes it. → captured in TECHNICAL_ARCHITECTURE.md known-gaps + design principles
- **Code reorg target layout is documented but not yet executed.** Docs forward-reference the target structure (`application/{ai,dashboard,data/fetchers,strategy,shared}/`); code remains flat in `application/` until a future session does the import refactor. → captured in TECHNICAL_ARCHITECTURE.md "Project Layout" section
- **Historical weather download removed from `weather.py`.** Only stadium-location extraction remains. NWS forecast is a stub function with v1-vs-v2 decision pending. → captured in data_sources.txt + weather.py docstring
- **Token vocabulary contract is no longer load-bearing for v1.** Moved to `_deferred/synthesis_pipeline/` along with the synthesis prompts.

1. **The `_deferred/synthesis_pipeline/` folder is intentional.** It's not "old code we forgot to delete." Read its README before assuming anything about it.
2. **Constitution-vs-code drift is partially fixed but not fully.** The docs now describe a target architecture that the code does not yet match. The "Known Gaps" sections in PRODUCT_ROADMAP and TECHNICAL_ARCHITECTURE call this out explicitly. Don't assume the file layout in docs reflects what exists on disk.
3. **The user's preference is explicit sparring-partner posture, not yes-and.** They've said it twice. Push back when warranted.
4. **The `application/data/` folder is gitignored, including subfolders.** The current code uses it for caches; v1 will add `snapshots/` and `advisor_log/` subdirectories.
5. **Historical weather is intentionally absent from v1.** Old `weather.py` had it; it was removed. NWS forecast is a stub.
6. **The user's existing `advisor.py` is a working prototype, not "the v1."** Phase 3 evolves it; doesn't replace it wholesale.
